# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.3.x   | :white_check_mark: |
| < 0.3   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please **do not** open
a public GitHub issue, as this may expose the vulnerability before it can be fixed.

Instead, report it privately using one of the following methods:

- **GitHub Private Vulnerability Reporting** (preferred):
  Use the [Report a vulnerability](../../security/advisories/new) button
  in the Security tab of this repository.
- **Email**: <ece.abdelrahman@gmail.com>

## Disclosure Policy

This project follows **coordinated vulnerability disclosure** practices:

1. You report the vulnerability privately.
2. We acknowledge receipt within **7 days**.
3. We investigate and aim to release a fix within **90 days** of the report.
4. Once a fix is released, the vulnerability details may be publicly disclosed.

## What Counts as a Vulnerability?

Examples of vulnerabilities include:

- Remote code execution
- Unauthorized data access or exfiltration
- Authentication or authorization bypass
- Injection flaws (SQL, command, etc.)

General bugs that do not have a security impact should be reported
via the normal GitHub Issues tracker.
